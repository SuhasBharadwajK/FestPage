C Programming Challenge
Rules:
  • All the participants have to register and get acceptance from the contest organizers.
  • Maximum participants per team is 2.
  • All the students irrespective of branch and year are eligible.
  • All the participants from a team should belong to the same college.
  • Every participant should bring his/her ID card.
  • Laptops and mobile phones are not allowed.
  • No external communication is allowed during the event like, phone calls, chat, email, browsing the Web etc.
  • No extra time will be given.
  • Team using any unfair means or breaking the discipline will be disqualified on the spot.
  • Judges' decision will be final.
  • There will be total of 2 rounds – Prelims and Mains
  • First round will test your basic knowledge of C.
  • Second Round will test your problem solving skills through C.
ROUND 1 (Prelimanary):
  • This will be a written elimination round.
  • The participating teams will be provided with question papers.
  • The goal is to estimate the correct output of the given code snippet.
  • No computers will be allotted in this round.
  • Duration for this round is 40 minutes.
  • 1 mark will be awarded for a correct answer.
  • The result of this round will be declared after 15 minutes.
ROUND 2 (Main Contest):
  • Teams will be given problems that are to be coded in C.
  • Duration for this round is 1 hour and 45 minutes.
  • Only complete execution with desired output is considered correct.
  • Each question will carry different weightage of marks, which will be mentioned in the question paper.
  • The result of this round will be declared after 20 minutes.

Web Design Contest
Rules:
  • Overview of the subject should be provided in one or two paragraphs.
  • Web site must be easy to navigate.
  • Use of frames is strongly not recommended.
  • The team should choose a team name.
  • All tags must be in lower case.
  • The document tags must be well formed.
  • All images should be coded using the 'alt' attribute.
  • Text sizes should be adaptive to the browser size.
  • The first page of the website must be 'index.html'.
  • All sub files and resources must reside in the same folder.
  • Web pages must be fully functional across all broswers.
  • Frames may be used, but should be made readable to Lynx.
  • If JavaScript is required for navigation, a non-script alternative navigation option must be included.
Judging Criteria
  • Content
    • Relavance of the content.
    • Accuracy of the content.
    • Correct spelling and grammar.
    • Good use of content chunking.
  • Aesthetics
    • Visual appeal and style.
    • Consistent look and feel.
    • Readability.
    • Minimalist design.
  • Design
    • Adherence to the HTML standards.
    • Effective navigation and working links.
    • Effective use of multimedia and animations.
    • Browser compatibility.
  • Usability and Accessibility
    • Availablity in users' native language.
    • Minimization of load on users' memory.
    • Minimization of required user actions.
    • Progressive exposition of levels of detail.
    • Navigation feedback mechanism.
    • Effective user guidance (help and error messages).
    • Accessibility for visually impaired users.
