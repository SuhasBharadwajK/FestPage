Web Design Contest
Rules:
  • Overview of the subject should be provided in one or two paragraphs.
  • Web site must be easy to navigate.
  • Use of frames is strongly not recommended.
  • The team should choose a team name.
  • All tags must be in lower case.
  • The document tags must be well formed.
  • All images should be coded using the 'alt' attribute.
  • Text sizes should be adaptive to the browser size.
  • The first page of the website must be 'index.html'.
  • All sub files and resources must reside in the same folder.
  • Web pages must be fully functional across all broswers.
  • Frames may be used, but should be made readable to Lynx.
  • If JavaScript is required for navigation, a non-script alternative navigation option must be included.
Judging Criteria:
  • Content
    • Relavance of the content.
    • Accuracy of the content.
    • Correct spelling and grammar.
    • Good use of content chunking.
  • Aesthetics
    • Visual appeal and style.
    • Consistent look and feel.
    • Readability.
    • Minimalist design.
  • Design
    • Adherence to the HTML standards.
    • Effective navigation and working links.
    • Effective use of multimedia and animations.
    • Browser compatibility.
  • Usability and Accessibility
    • Availablity in users' native language.
    • Minimization of load on users' memory.
    • Minimization of required user actions.
    • Progressive exposition of levels of detail.
    • Navigation feedback mechanism.
    • Effective user guidance (help and error messages).
    • Accessibility for visually impaired users.
